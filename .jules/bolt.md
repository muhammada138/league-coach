## 2024-03-24 - React Main Thread Blocking and Memoization Anti-Patterns
**Learning:** Found an anti-pattern in the codebase where computationally heavy array filtering (`.reduce`, `.filter`) and synchronous I/O operations (`localStorage.getItem`) were left exposed in standard render flows (such as `SummaryStrip`, `TeammatesContent`, and `Dashboard.jsx`). Without explicit `useMemo` hooks with strict dependency arrays, rapid visual updates (like hovering over an element that updates state quickly via `onMouseEnter`) would force these functions to redundantly re-run, dropping frames and spiking CPU utilization. Note that this causes actual "jank" and is specifically warned against in the AGENTS.md document under 'React Performance Pattern'. Furthermore, ensuring that proper dependency arrays are used prevents triggering the React Compiler logic for dropping memoizations.
**Action:** Always wrap heavy data aggregations or synchronous I/O inside components with `useMemo`, and closely adhere to dependency arrays (e.g. `[games, tab]`). Avoid letting `.map()` or `.filter()` arrays fall into the top level body of the function component if it can be avoided.
