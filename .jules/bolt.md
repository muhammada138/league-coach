## 2026-04-21 - Prevent synchronous localStorage I/O on hover
**Learning:** In `LPGraph`, checking `localStorage` (via `JSON.parse`) on every render caused synchronous I/O blocks during rapid hover events, slowing down interactions and causing unnecessary re-renders. A similar issue was identified in `LpHistoryGraph` with object mapping on every render. `SearchInput` was also recalculating suggestions on every arrow-key press since it depended on `focusedIdx` unintentionally through closures instead of a dependency array.
**Action:** Always wrap data computations involving synchronous I/O or expensive mapping in `useMemo` with explicit dependency arrays. Ensure UI state changes like `focusedIdx` or `hoveredIdx` do not trigger full recalculations of static or cached datasets.
